# الفصل الأول: المقدمة (Introduction)

## 1.1 خلفية المشكلة
تُعدّ الزراعة ركيزةً أساسية للاقتصاد الفلسطيني، إذ تشكّل مصدر رزق لشريحة واسعة من السكان في مناطق مثل نابلس وطولكرم وأريحا. غير أن أمراض النباتات تُلحق خسائر فادحة بالمحاصيل سنوياً؛ فوفقاً لمنظمة الأغذية والزراعة (FAO)، تتراوح الخسائر الناجمة عن الأمراض النباتية بين 20% و40% من إجمالي الإنتاج الزراعي العالمي.

المشكلة الجوهرية تكمن في التشخيص المتأخر: يعتمد المزارعون عادةً على التجربة الشخصية أو الانتظار حتى تتفاقم الأعراض، مما يُفضي إلى خسائر كبيرة كان يمكن تفاديها بتشخيص مبكر ودقيق.

## 1.2 أهمية المشروع
- **اقتصادياً**: تقليل الخسائر الزراعية وزيادة الإنتاجية
- **بيئياً**: تقليل الاستخدام العشوائي للمبيدات
- **اجتماعياً**: دعم صغار المزارعين بأداة مجانية وسهلة الاستخدام

## 1.3 أهداف المشروع
1. بناء نموذج ذكاء اصطناعي يكشف أمراض النباتات من الصور بدقة ≥ 90%
2. تطوير تطبيق ويب سهل الاستخدام يعمل على الهاتف المحمول
3. دمج شات بوت زراعي ذكي لتقديم نصائح مخصصة
4. توفير قاعدة بيانات شاملة للأمراض الشائعة باللغة العربية

# الفصل الثاني: الخلفية ومراجعة الأدبيات (Background & Literature Review)

## 2.1 الذكاء الاصطناعي في الزراعة
شهدت السنوات الأخيرة تحولاً جذرياً في تطبيقات الذكاء الاصطناعي بالقطاع الزراعي. تقنية Computer Vision تُمكّن الآلات من "رؤية" وتحليل الصور بدقة تفوق القدرة البشرية في بعض الحالات.

## 2.2 الدراسات السابقة
| الدراسة | النتيجة |
|---------|---------|
| Hughes & Salathé (2015) - PlantVillage | دقة 99.35% باستخدام CNN على 54,306 صورة |
| Mohanty et al. (2016) | دقة 87.5% في بيئات حقيقية |
| Brahimi et al. (2017) | تشخيص أمراض الطماطم بدقة 99.18% |

## 2.3 تقنيات الذكاء الاصطناعي المستخدمة

### Convolutional Neural Networks (CNN)
شبكات عصبية اصطناعية مصممة خصيصاً لمعالجة الصور. تتعلم استخراج الأنماط البصرية تلقائياً.

### Transfer Learning
بدلاً من بناء نموذج من الصفر، نستخدم نموذج MobileNetV2 المدرب مسبقاً على ملايين الصور، ثم نكيّفه لمهمتنا. هذا يوفر:
- وقت تدريب أقل
- دقة أعلى مع بيانات أقل
- كفاءة في الموارد الحسابية

### Five AI Concepts في مشروعنا
1. **Perception (الإدراك)**: الكاميرا ترى صورة الورقة
2. **Representation & Reasoning (التمثيل والاستدلال)**: النموذج يحلل الأنماط ويستنتج المرض
3. **Machine Learning (التعلم الآلي)**: النموذج يتعلم من 54,305 صورة
4. **Natural Interaction (التفاعل الطبيعي)**: الشات بوت يتحدث بالعربية
5. **Societal Impact (الأثر المجتمعي)**: مساعدة المزارعين وتقليل الخسائر

## 2.4 Dataset - PlantVillage
- **المصدر**: Penn State University - مفتوح المصدر
- **الحجم**: 54,305 صورة
- **الفئات**: 38 فئة (مرض وسليم) في 14 نوع نبات
- **الجودة**: صور عالية الدقة في بيئات خاضعة للسيطرة

# الفصل الثالث: تصميم الحل (Solution Design)

## 3.1 هيكل النظام

```
[المزارع] → [يلتقط صورة] → [يرفعها للتطبيق]
                                    ↓
[نموذج AI يحلل الصورة] → [يُحدد المرض + نسبة الثقة]
                                    ↓
[عرض النتيجة + العلاج + الوقاية] → [شات بوت للأسئلة]
```

## 3.2 مكونات النظام

### 3.2.1 نموذج التشخيص (AI Model)
- **الأساس**: MobileNetV2 (Transfer Learning)
- **حجم الصورة**: 224×224 بكسل
- **طبقات إضافية**: GlobalAveragePooling → Dropout(0.3) → Dense(128) → Output
- **دالة الخسارة**: Categorical Crossentropy
- **المُحسِّن**: Adam (lr=0.001)

### 3.2.2 تطبيق الويب
- **الخادم**: Python Flask
- **الواجهة**: HTML/CSS/JavaScript
- **التواصل**: REST API (JSON)

### 3.2.3 الشات بوت الزراعي
- **المحرك**: Claude AI (Anthropic API)
- **السياق**: يتلقى نتيجة التشخيص ويجيب على أسئلة مرتبطة
- **اللغة**: العربية حصراً

## 3.3 قاعدة بيانات الأمراض
تحتوي على 11 مرض موثق بالعربية يشمل:
- اسم المرض (عربي/إنجليزي)
- مستوى الخطورة
- سبب المرض (فطر/بكتيريا/حشرة)
- الأعراض
- طريقة العلاج
- الوقاية

## 3.4 خطوات التطوير
1. **جمع البيانات**: تحميل PlantVillage Dataset
2. **معالجة البيانات**: تصغير الصور + augmentation
3. **تدريب النموذج**: Transfer Learning + Fine-tuning
4. **بناء الواجهة**: HTML/CSS/JS متجاوب
5. **دمج الشات بوت**: Claude API
6. **الاختبار**: اختبار بصور حقيقية من الحقل

# الفصل الرابع: تحليل البيانات والنتائج (Data Analysis & Results)

## 4.1 إحصائيات Dataset
| المعطى | القيمة |
|-------|-------|
| إجمالي الصور | 54,305 |
| صور التدريب (80%) | 43,444 |
| صور التحقق (20%) | 10,861 |
| عدد الفئات | 38 |
| أنواع النباتات | 14 |

## 4.2 نتائج التدريب (متوقعة)
| المقياس | القيمة |
|---------|-------|
| دقة التدريب | ~97% |
| دقة التحقق | ~93-95% |
| خسارة التحقق | ~0.15-0.25 |

## 4.3 أبرز الأمراض في Dataset
1. Tomato Late Blight - 1909 صورة
2. Tomato Early Blight - 1000 صورة
3. Potato Late Blight - 1000 صورة
4. Pepper Bacterial Spot - 997 صورة

## 4.4 تقييم النموذج
- **Precision**: دقة التشخيص الإيجابي
- **Recall**: نسبة اكتشاف الحالات الفعلية
- **F1-Score**: التوازن بين Precision و Recall
- **Confusion Matrix**: تحليل الأخطاء بين الفئات
